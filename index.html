Cypress.io

Pour Contre
Capture d'écran pour chaque étape du script Pas de support multi-navigateur
Utilisation de Mocha comme fournisseur de structure Les tests en parallèle manquent encore
Pas d'utilisation de Selenium
  • Cypress n’utilise pas de sélénium: la plupart des outils de bout en bout que nous avons expérimentés utilisent du sélénium, c’est pourquoi ils ont presque les mêmes problèmes.
  • Cypress supporte très bien n'importe quel framework ou site web: Il existe des centaines de projets utilisant les derniers frameworks React, Angular, Vue, Elm, etc. Cypress fonctionne également aussi bien sur les anciennes pages de rendu de serveur ou les applications.
  • Les tests Cypress sont uniquement écrits en JavaScript: Bien que vous puissiez compiler en JavaScript depuis n’importe quel autre langage, le code de test est finalement exécuté dans le navigateur lui-même. Il n'y a pas de langues ou de liaisons de pilotes - il y a et il n'y aura que JavaScript.
  • Il n’ya pas de dépendances, vous mettez votre test dans package.json et c’est tout.
  • Cypress est beaucoup plus rapide en comparaison avec les outils de bout en bout de Selenium que nous avons expérimentés.
  • Il y a une capture d'écran pour chaque étape de votre script, ce qui peut être très utile en cas de test de réussite ou d'échec, c'est bon pour le débogage !!
  • Cypress a une cyntaxe claire, il est facile à lire, vous l’aimerez!
  • La structure étant différente des autres outils de bout en bout de Selenium, vous devrez peut-être tout d'abord passer plus de temps à comprendre la structure et à trouver le meilleur moyen de créer vos scripts.
  • Caractéristiques. Aucun support de téléchargement de fichier. Pas de tests de navigateurs croisés. Qui sait quand ces problèmes seront traités? Pour les grands projets, ces fonctionnalités sont essentielles.
  • Modèle d'objet de page. C'est quelque chose qui a déjà été prouvé par le temps. Cypress soutient une approche différente qui pourrait être controversée. Plus de détails à ce sujet sont ici: Cypress POM
  • Il est uniquement disponible pour un seul client (langue), c'est-à-dire pour JavaScript uniquement. Donc, pour travailler avec cela, vous devez connaître JavaScript. Cependant, cela pourrait être un avantage pour l’application JavaScript, mais j’aimerais le mettre comme un inconvénient pour ceux qui ont des difficultés avec javascript.

NightwatchJS

Pour Contre
Syntaxe propre Support de Mocha uniquement
Intégration continue
Prise en charge de CSS et Xpath
  • Syntaxe propre: une syntaxe simple mais puissante vous permet d'écrire des tests très rapidement.
  • Programme d'exécution de test intégré: le programme d'exécution de test de ligne de commande intégré peut exécuter les tests de manière séquentielle ou parallèle, ensemble, par groupe, balises ou une seule fois.
  • Prise en charge des services cloud: fonctionne avec des fournisseurs de tests cloud, tels que SauceLabs et BrowserStack.
  • Prise en charge de CSS et Xpath: Des sélecteurs CSS ou Xpath peuvent être utilisés pour localiser et vérifier des éléments sur la page ou pour exécuter des commandes.
  • Prise en charge de l'intégration continue: les rapports XML JUnit sont intégrés pour vous permettre d'intégrer vos tests dans votre processus de construction à des systèmes tels que Teamcity, Jenkins, Hudson, etc.
  • Il n’a pas beaucoup de choix pour les frameworks de tests unitaires car il dispose de son propre framework de tests et supporte également Mocha.
  • Un support légèrement inférieur à WebDriverIO et à Protractor.
Exemple avec Cypress

describe('My First Test', function() {
    it('clicking "type" navigates to a new url', function() {
        cy.visit('https://example.cypress.io')
    
        cy.contains('type').click()
    
        cy.url().should('include', '/commands/actions')
    })
})
Exemple avec NightwatchJS

module.exports = {
    'Demo test Google' : function (browser) {
        browser
        .url('http://www.google.com')
        .waitForElementVisible('body')
        .setValue('input[type=text]', 'nightwatch')
        .waitForElementVisible('input[name=btnK]')
        .click('input[name=btnK]')
        .pause(1000)
        .assert.containsText('#main', 'Night Watch')
        .end();
    }
};
